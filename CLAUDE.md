## Project Overview

EasyPOS is a Java 8 Swing desktop Point-of-Sale application built with NetBeans. It targets small retail businesses in Sri Lanka with multi-language support (Sinhala, Tamil, English).

## Build and Run

This is an Apache Ant project managed by NetBeans. From the project root:

```
ant build        # compile sources
ant run          # compile and run main class (easyPOS.OpenScreen)
ant jar          # produce dist/EasyPOS.jar
ant clean        # remove build/ and dist/
```

To run a single test class:
```
ant test -Dtest.class=<fully.qualified.ClassName>
```

The primary way to develop is via NetBeans IDE (open `nbproject/project.xml`). The main entry point is `easyPOS.OpenScreen`.

## Architecture

### Package structure

| Package | Role |
|---|---|
| `easyPOS` | Swing UI panels organized by feature (sale/, stock/, customers/, supplier/, reports/, settings/, login/, user/, invoice/) |
| `control` | Application-level singletons and infrastructure |
| `dataModels` | Local-only data models (enums, simple POJOs) |
| `appDataModels` | App runtime models (RuntimeData, Permission, API response shapes) |
| `serverDataModels` | Mirror models for server-side entities (Category, Item, SaleInvoice, etc.) |
| `dbOperations` | Legacy MySQL database operations |
| `localDatabase` | SQLite local database via `DatabaseManager` |
| `webService` | HTTP client layer for the remote PHP API |
| `tableModels` | Swing `TableModel` implementations |
| `uiUtil` / `util` | Shared UI and general utilities |

### Core singletons in `control`

- **`ApplicationDataManager`** — persistent app settings: language configuration (Sinhala/Tamil/English per receipt section), thermal printer name, invoice number counter, receipt logo. Call `getInstance()`.
- **`RuntimeDataManager`** — session state: auth token, institute ID, terminal ID, currently loaded categories, logged-in user. Call `getInstance()`.
- **`EventManager`** — in-process event bus. Events: `LoginEvent`, `MenuItemChangeEvent`, `SalesMenuItemClickListener`, `NumberPadKeyPressListener`. Components register listeners; others call `notify*()` methods.
- **`EasyPosLogger`** — async file logger writing to `C:/EasyPOS/Logs/application-YYYY-MM-DD.log` (30-day retention). Use `getInstance().log(LogLevel.INFO/DEBUG/ERROR, message)`.
- **`ServerDataSubmissionQueue`** — background thread queue that serializes all outbound API calls; tasks are submitted as `Runnable`s and executed one at a time by a daemon thread.

### Dual database strategy

The app has two local databases and one remote API:

1. **SQLite** (`myapp.db` in working directory) — primary offline store. Managed by `localDatabase.DatabaseManager` singleton. Tables: `user_account`, `user_permission`, `category`, `change_log`, `measure_unit`, `institute`, `app_data`, `item`, `item_stock`.
2. **MySQL** (`jdbc:mysql://localhost:3306/pharmacy_management_system`) — legacy local store. Credentials in `dbOperations.DBCons`. Used by classes in `dbOperations/`.
3. **Remote PHP API** (`https://moosero.lk/EASYPOS-API-PHP-master/api`) — cloud sync. All calls go through `webService.WebService` (raw HTTP) and `webService.ServerAPIConnection` (typed methods). Auth token from `RuntimeDataManager` is sent as a request header. Offline queuing is handled by `ServerDataSubmissionQueue`.

### UI navigation

`easyPOS.OpenScreen` is the root `JFrame`. It contains a `jPanelBase` container that swaps between a login panel and a main menu panel. Navigation between feature areas (Sales, Stock, Customers, etc.) is driven by `MenuItemType` enum values dispatched through `EventManager.notifyMenuItemChanged()`.

### Transaction ID format

Transaction IDs follow: `INSTN(4)-TERM(6)-DATE(8)-SEQ(5)` e.g. `000100000120250920 00001`. Generated by `ApplicationDataManager.getNextTransactionId()`.

### Runtime file paths (hardcoded)

- `C:/EasyPOS/Images/Category/` — category images
- `C:/EasyPOS/Images/Item/` — item images
- `C:/EasyPOS/Images/Other/` — receipt logos
- `C:/EasyPOS/Logs/` — application logs

### Key third-party libraries

- **JTattoo 1.6.11** — Swing Look and Feel (Aluminium theme applied in `OpenScreen`)
- **iText 5.5.5** — PDF generation for reports and receipts
- **JFreeChart 1.0.19** — charts in `ReportPanel`
- **JasperReports 5.0.1** — report templates
- **Jackson 2.9.9** — JSON serialization for API calls
- **SQLite JDBC 3.50.3.0** — local SQLite database
- **JCalendar 1.4** — date picker component

### Hardware integrations

- **BIXOLON SRP-E302** thermal printer — receipt printing via `SimpleReceiptPrint` and `SimpleCustomerStatementPrint`
- **Zebra label printer** — barcode sticker printing via `ZebraStickerPrinter`
- Receipt printer name is configurable via `ApplicationDataManager.setReceiptPrinterName()`

### Localization

UI strings are in `easyPOS/Bundle.properties` (English default) and `easyPOS/Bundle_si_LK.properties` (Sinhala). The active `ResourceBundle` is retrieved from `ApplicationDataManager.getResourceBundle()`. The `Language` enum has values `ENGLISH`, `SINHALA`, `TAMIL`. Receipt sections (header, items, footer, business data) each have an independent language setting.
